Introduction to Khroma

Khroma is a next-generation documentation engine that adapts to your codebase in real-time. It uses advanced vector embeddings to understand not just what your code does, but why it exists.

The Paradigm Shift

Traditional documentation is static. It rots the moment it's written. Khroma changes this by treating documentation as a living stream of intelligence derived directly from your CI/CD pipeline.

By integrating our deeply-coupled SDK, every commit allows Khroma to re-evaluate the semantic meaning of your architecture.

Installation

Get started in seconds. Our CLI handles the heavy lifting of indexing your repository and setting up the initial knowledge graph.

bash 
# Install the Khroma CLI globally
npm install -g @khroma/cli

# Initialize in your project root
khroma init --mode=strict

# > Scanning repository... Done (0.4s)
# > Generating vector embeddings... Done (2.1s)
# > Khroma is ready at http://localhost:9900

Configuration

Khroma is highly configurable via the khroma.config.js file. You can define custom parsers, exclude paths, and tune the AI temperature for generated summaries.

javascript 
export default {
  project: 'omega-core',
  theme: 'dark-premium',
  ai: {
    temperature: 0.2,
    model: 'gpt-4-turbo-preview'
  }
}
